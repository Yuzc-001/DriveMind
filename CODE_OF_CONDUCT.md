# Code of Conduct

DriveMind aims to be a calm, respectful project.

## Expected behavior
- be respectful, patient, and clear
- assume good intent while staying honest about problems
- focus criticism on ideas, code, docs, and behavior
- make room for newcomers and basic questions
- respect project boundaries around safety, licensing, and branding.

## Unacceptable behavior
- harassment, threats, or intimidation
- discriminatory or demeaning language
- repeated bad-faith disruption
- sharing private information without permission
- trying to win arguments by belittling contributors.

## Enforcement
Project maintainers may remove comments, close threads, or restrict participation when behavior undermines a healthy working environment.

## Reporting
If you experience or observe unacceptable behavior, open a maintainer-facing issue or request a private follow-up path through the repository.
