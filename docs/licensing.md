# Licensing Guide

## Current model
DriveMind now uses a standard **source-available** model instead of a custom usage notice.

The project is licensed under **Business Source License 1.1 (BUSL-1.1)** with:
- **Additional Use Grant:** None
- **Change Date:** March 11, 2030
- **Change License:** GPL-2.0-or-later

## Why this model
This is a better fit than the previous custom notice because it is:
- easier for developers and companies to recognize
- clearer about non-production vs production use
- explicit about future conversion to an Open Source license
- more credible for a serious public repository

## What is allowed now
Until the Change Date, BUSL-1.1 allows:
- reading and auditing the source
- internal evaluation and experimentation
- modification and redistribution under the same BUSL-1.1 terms
- non-production use

## What is not allowed now
Until the Change Date, production use requires separate permission from the licensor.
That includes using DriveMind as part of a live product, service, or operational production workflow.

## What changes on March 11, 2030
On March 11, 2030, or the fourth anniversary of the first public distribution of a specific covered version under BUSL-1.1, whichever comes first, that version converts to **GPL-2.0-or-later**.

## Open source vs source-available
DriveMind should not currently be described as Open Source.
The accurate wording today is:
- `source-available under BUSL-1.1`
- `scheduled to convert to GPL-2.0-or-later on March 11, 2030`

## Brand boundary
The code and docs are licensed under BUSL-1.1.
The DriveMind name and logo remain brand assets and are not granted for general trademark use by default.

## Recommended public wording
- `DriveMind is source-available under BUSL-1.1 and scheduled to convert to GPL-2.0-or-later on March 11, 2030.`
- `Production use before March 11, 2030 requires separate permission.`
