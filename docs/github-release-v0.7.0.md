# DriveMind v0.7.0

DriveMind v0.7.0 is the execution-ceiling release.

## Highlights

- Defines DriveMind around execution integrity, execution ceiling, and experience compounding.
- Adds `skill/references/execution-ceiling.md`.
- Rebuilds the active repository surface as a v0.7-only product.
- Replaces the primary README visual with an image2.0-generated Signal Gate lockup.
- Hardens install and bootstrap scripts with package validation and clean replacement.
- Removes non-current release notes, redirect references, and outdated examples from the current tree.

## Core change

DriveMind now asks:

# is the model settling below its real execution capability?

When the current result is too shallow and a safe stronger move exists, DriveMind should force one concrete improvement before settling.

## Boundary

Execution ceiling does not authorize reckless persistence. Decision boundaries still override execution intensity.
