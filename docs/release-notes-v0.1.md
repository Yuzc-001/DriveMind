# DriveMind v0.1 Foundation Release

## Release status
This is the first foundation release of DriveMind. It packages the project into a reusable repository with a clear product story, operating model, initial skill, install path, standard source-available licensing, governance docs, validation workflow, and first visual identity assets.

## What ships now
- the first reusable DriveMind skill
- one-click GitHub bootstrap entry points
- Codex and Claude Code install helpers
- BUSL-1.1 source-available licensing with a March 11, 2030 GPL change date
- persistence, escalation, and mode references
- diary, review, and distillation templates
- a first logo system with SVG source assets and local raster export tooling
- a canonical brand kit plus activation, installation, interaction, and licensing docs
- governance docs for contributions, conduct, security, release history, and trademark boundaries
- a minimal validation workflow for scripts and markdown links
- examples showing DriveMind with vs without the layer
- promo copy and release assets for public sharing.

## What changed in this iteration
- repositioned DriveMind around calm reliability, safety boundaries, and human collaboration
- collapsed repeated copy docs into one canonical brand kit
- reshaped the README into a clearer homepage-style entry point
- unified roadmap language under `v0.x`
- renamed the strongest mode language to stay consistent with the product voice
- added portable personal and project install modes
- added local installers and remote one-click bootstrap commands
- replaced the custom usage notice with a standard BUSL-1.1 source-available model
- added governance docs and a minimal CI validation workflow
- added the first DriveMind logo system and visual identity guidance.

## Why this matters
`v0.1 Foundation` is the point where DriveMind stops reading like a loose idea and starts behaving like a product foundation. The repository now shows what DriveMind is, how it should feel, what it ships, how to install it, how it is licensed, how it is governed, how it looks as a product, who it is for, and how it can evolve.

## Still not in scope
- runtime integration with a live agent stack
- dashboard or hosted service
- automated runtime evaluation suite
- broad commercial packaging
- a full visual identity system.

## Next
- sharper retry heuristics
- better failure signatures and memory categories
- tighter activation patterns for tool-using agents
- broader installation coverage across agent ecosystems
- inverse, monochrome, and social variants of the logo.
