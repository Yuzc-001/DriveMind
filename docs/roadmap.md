# DriveMind Roadmap

## `v0.1 Foundation`
- define execution modes
- generate daily diary
- generate post-task review
- preserve reusable lessons
- package the first behavior skill
- support portable installation across Codex and Claude Code
- establish licensing, governance docs, and a minimal validation workflow.

## `v0.2 Validated`
- stronger natural-language triggering for calm persistence and boundary-aware instructions
- `Intensive` mode naming and clearer reporting expectations
- structured-but-adaptive review behavior
- review style guidance for more natural retrospectives

## `v0.3 Formal Release`
- lightweight task typing for judgment, boundary, progress, diagnosis, and distillation tasks
- explicit decision gates for high-impact actions, production/release decisions, external representation, and data/deletion work
- stuck-resolution guidance so persistence becomes blocker diagnosis and momentum recovery
- compression rule to keep short answers natural and avoid visible framework over-expansion

## `v0.4 Context Intelligence`
- session handoff protocol for cross-session continuity without burdening short tasks
- confidence signaling reference for structured uncertainty communication
- language consistency: all skill references now in English
- zero-config auto-detect install: detects Claude Code and Codex, installs to all found targets

## Product principle
DriveMind should mature as a calm reliability layer and collaboration doctrine for agents working in long-lived real tasks.

Its deeper direction is simple:
- clear goals
- clear boundaries
- enough freedom to act

Not an operations manual.
